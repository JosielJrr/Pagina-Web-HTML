Evolução dos Computadores

(voltar)
  1. Primeira Greação:
    • Havard Mark I (1944):

      • Embora a ideia do computador já fosse concebida nas primeiras máquinas de cálculo, o primeiro artefato que marca o início da primeira geração é o Harvard Mark I, desenvolvido para processar cálculos mais complexos.

    • Colossus (1946):

      • O Colossus foi um computador desenvolvido pelo Reino Unido no período da Segunda Guerra Mundial para criptoanálise*, sendo utilizado para decifrar códigos alemães.

    • Eniac (1946):

      • Nos Estados Unidos, o ENIAC, desenvolvido durante a Segunda Guerra Mundial, foi um marco na história da computação, sendo o primeiro computador eletrônico digital de grande escala.

  2. Segunda Geração:

    3. A segunda geração dos computadores, ocorrida aproximadamente entre 1959 e meados da década de 1960, testemunhou a transição das válvulas eletrônicas para os transistores. Esse avanço tecnológico resultou em computadores mais eficientes e compactos.

  3. Terceira Geração:

    4. A terceira geração de computadores, que se estendeu de 1965 a 1970, foi caracterizada pela introdução dos circuitos integrados, também conhecidos como chips. Esses chips consolidaram transistores, capacitores e resistores em um único componente. Esse avanço tecnológico permitiu a criação de sistemas mais compactos e poderosos, marcando uma fase crucial na evolução da computação.

  4. Quarta Geração:

    5. A quarta geração teve início em 1971 e persiste até os dias atuais. Durante esse período, os circuitos integrados evoluíram para microprocessadores, representando uma transição significativa. Os chips continuam sendo fundamentais, mas em versões mais avançadas. Esta geração é marcada pelo surgimento dos PCs (Computadores Pessoais), contribuindo para a popularização da tecnologia. Nesse contexto, destaca-se também a introdução do primeiro mouse, associado à Apple, como parte integrante dos avanços na interação homem-computador.

  5. Quinta Geração?

    6. Muitas pessoas levantam a hipótese de estarmos ingressando na quinta geração dos computadores, dado o cenário atual de avanços tecnológicos. A ascensão da computação quântica, o impacto das criptomoedas, o metaverso e o contínuo desenvolvimento da inteligência artificial movem questões sobre a necessidade de reavaliar as tradicionais classificações em gerações. Enquanto alguns observadores consideram essas inovações como indicadores de uma nova era na evolução dos computadares, a sua presença ainda não é amplamente definida.

O que são Clients?

(voltar)

Em linguagem web, o termo "clients" refere-se aos dispositivos que consomem informações da internet, como notebooks e celulares. Esses dispositivos recebem dados dos servidores por meio da rede mundial de computadores.

Quando esses dispositivos acessam um site, a interação ocorre por meio da internet: a solicitação é enviada ao servidor, que por sua vez retorna arquivos HTML, CSS, JavaScript, imagens, entre outros. Esses arquivos são denominados estáticos. A eficiência do acesso do usuário é influenciada pela velocidade da conexão e pelo tamanho dos arquivos recebidos, sendo ambos fatores cruciais para uma experiência de navegação otimizada.

Os "clients" desempenham um papel central no ecossistema digital, especialmente quando consideramos os navegadores, aplicações web e dispositivos móveis. Navegadores são interfaces cruciais que facilitam a interação entre usuários e a vasta gama de aplicações web disponíveis. Essas aplicações, por sua vez, são projetadas para serem acessadas por meio de navegadores, proporcionando funcionalidades diversas, desde plataformas de mídia social até serviços de e-commerce. Diante desse contexto, os dispositivos móveis têm se tornado cada vez mais predominantes, tornando o acesso à internet e aplicativos ainda mais popular. O design responsivo, especialmente crucial em dispositivos móveis, assegura uma experiência adaptável em diversas telas, otimizando a usabilidade. Essa interconexão entre "clients", navegadores, aplicações web e dispositivos móveis cria um ambiente digital dinâmico, influenciando a maneira como os usuários interagem e exploram o vasto universo online.

O que são Servers?

(voltar)

Os servidores, ou "servers", desempenham um papel crucial no cenário digital. Existem servidores amplamente acessados através da internet, servindo como pilares para a conectividade global. No entanto, também existem servidores que operam em um ambiente mais restrito, sendo utilizados localmente, muitas vezes em empresas, sem depender da internet (intranet). Os servidores, geralmente, são configurados como computadores mais robustos e caros, uma vez que são projetados para lidar com uma capacidade substancial de processamento de dados. Além disso, é importante destacar que a diversidade de servidores é notável. Eles podem variar desde computadores mais simples, utilizados para armazenar dados em menor escala, até estruturas mais complexas, como data centers, que oferecem capacidade de processamento e armazenamento em grande escala.

Softwares em servidores:

Quando um cliente emite uma requisição HTTP para um servidor, a solicitação é inicialmente recebida pelo sistema operacional do servidor. O sistema operacional desempenha um papel crucial no gerenciamento dos recursos do servidor, como CPU, memória e armazenamento. Funcionando como uma camada intermediária, ele recebe as solicitações e as encaminha para o servidor web correspondente. Diversos sistemas operacionais são dedicados a servidores, incluindo o Windows Server, desenvolvido pela Microsoft, e várias distribuições baseadas em Linux, como Ubuntu, CentOS, Fedora, Debian e Oracle. Essas opções oferecem flexibilidade para atender às necessidades específicas de diferentes ambientes de servidor.

Os servidores web, como Apache ou NGINX, são aplicativos especializados projetados para lidar com solicitações HTTP. Quando a requisição chega ao sistema operacional, este encaminha a solicitação para o servidor web correspondente, que então processa a requisição, determina como responder e, se necessário, interage com outros componentes, como bancos de dados ou aplicativos.

Tipos de Servidores:

(voltar)
  • Proxy:

    • O servidor proxy atua como intermediário no acesso dos dispositivos à internet, impondo regras como o bloqueio de determinados sites, sendo frequentemente utilizado em ambientes empresariais. Além disso, desempenha a função de servidor de autenticação, controlando o acesso com base em credenciais. . Em algumas situações, é possível que, ao realizar uma requisição HTTP, os dados passem pelo proxy, sejam processados pelo servidor central e, ao retornarem, sejam bloqueados pelo próprio proxy.

  • Firewall:

    • O firewall, embora não seja estritamente um servidor, desempenha uma função fundamental na garantia da segurança de dados, servindo como uma barreira protetora contra acessos externos não autorizados. Sua principal responsabilidade é monitorar e controlar o tráfego de rede, baseando-se em um conjunto predefinido de regras. Essa funcionalidade é essencial para proteger redes corporativas contra ameaças, como o envio de arquivos maliciosos, fornecendo uma camada adicional de segurança contra atividades indesejadas ou potencialmente prejudiciais.

  • Web server:

    • Um servidor web é um software projetado para gerenciar solicitações de clientes e fornecer respostas correspondentes, principalmente no contexto de acessar sites ou aplicativos pela internet. Ele atua como uma ponte entre os clientes e os recursos web, processando requisições, executando aplicativos e entregando conteúdo, como páginas da web, imagens e outros dados. Servidores web populares incluem o Apache e o NGINX. Esses componentes são essenciais para a infraestrutura da internet, garantindo a entrega eficiente e segura de informações aos usuários.

  • E-mail server:

    • Um servidor de e-mail, é projetado especificamente para armazenar, gerenciar e facilitar a troca de e-mails. Esse tipo de servidor lida com o recebimento, envio, armazenamento e encaminhamento de mensagens de e-mail.

  • Database server:

    • Em muitas aplicações web, é comum ter um servidor de banco de dados, que armazena e gerencia as informações dos usuários. As aplicações web instaladas no servidor podem acessar esse banco de dados para recuperar, modificar ou armazenar dados, garantindo a integridade das informações. Esse tipo servirdor desempenha um papel crítico na gestão eficiente e segura dos dados, sendo essencial para aplicações que dependem de armazenamento de informações, como sistemas de gerenciamento de conteúdo, comércio eletrônico, redes sociais e muito mais. Em muitas arquiteturas de aplicativos web, o servidor de banco de dados frequentemente opera como um componente separado, conectado a um servidor central. Essa separação de funções permite uma melhor organização e controle das operações, contribuindo para o desempenho eficaz e a manutenção da integridade dos dados.

  • Servidor DNS:

    • Um servidor DNS mantém uma tabela ou banco de dados que associa nomes de domínio (como "www.exemplo.com") a endereços IP específicos. Quando um usuário digita um nome de domínio em um navegador ou realiza qualquer outra solicitação na internet, o servidor DNS é consultado para traduzir esse nome de domínio em um endereço IP correspondente. Ele desempenha um papel crucial na navegação na web, pois permite que os usuários acessem sites digitando nomes de domínio amigáveis em vez de terem que memorizar os números de IP correspondentes.

Como é uma estrutura básica de um HTML?

(voltar)

O HTML é uma linguagem de marcação de hipertexto amplamente utilizada na criação de páginas web. Ao contrário de uma linguagem de programação, o HTML é voltado para a estruturação e organização do conteúdo. Para começar a estruturar um arquivo HTML, é fundamental iniciar com a tag < html > < /html >. Dentro dessa tag, é possível definir o cabeçalho da página entre as tags < head > < /head > e o corpo do arquivo, onde as informações visíveis para o usuário são adicionadas, entre as tags < body > < /body >. O HTML também oferece tags de formatação de texto, como < i >< /i > para itálico, < strong >< /strong > para negrito e < u >< /u > para sublinhado, aprimorando a apresentação visual e a legibilidade do conteúdo em páginas web.

A tag < input > é utilizada para permitir que os usuários adicionem informações em uma página web. Ao empregar o atributo type, é possível definir diferentes formatos. Por exemplo, < input type="text"/ > possibilita a inserção de informações em formato de texto, enquanto < input type="number"/ > aceita apenas valores numéricos. Além disso, a tag < img/ > é empregada para incorporar imagens na página, ela também é auto-fechável e pode ser configurada com atributos como src para indicar o endereço da imagem, width para especificar a largura desejada e alt para fornecer uma descrição textual da imagem. Um exemplo seria: < img width="150" src="endereço da imagem" alt="descrição"/ >. Esses elementos são essenciais para enriquecer a experiência do usuário ao interagir com o conteúdo da página web.

As tags voltadas para a estruturação de textos são conhecidas como elementos de tipografia em HTML. Para iniciar um título, utiliza-se < h1 >< /h1 >, sendo possível prosseguir até a tag < h6 >< /h6 >, que define subtítulos. A estrutura do texto é delineada pela tag < p >< /p >, onde cada parágrafo é encapsulado individualmente para proporcionar a quebra no conteúdo. No caso de citações, a tag apropriada é < blockquote >< /blockquote >.

Para inserir um link em um arquivo HTML, utiliza-se a tag < a >< /a >. Dentro dessa tag, o atributo href é empregado para especificar o endereço do link. O atributo target pode ser utilizado para indicar que o link deve abrir em uma nova aba do navegador, sendo possível utilizar target="_blank" para essa finalidade. Exemplo: < a href="endereço do link" target="_blank">Nome do link no arquivo< /a >.

Listas Ordenadas e Não Ordenadas

(voltar)

No HTML, há três dois tipos principais de listas para estruturar textos: as ordenadas, representadas pela tag < ol >< /ol >, e as não ordenadas, identificadas pela tag < ul >< /ul >. Dentro dessas tags, os itens individuais são adicionados por meio da tag < li >< /li >. Vale destacar que é possível incorporar uma lista dentro da outra, proporcionando uma hierarquia de listas.

Exemplos:

Lista Ordenada < ol >< /ol >:

  1. Item 1
  2. Item 2
  3. Item 3

Listas Não Ordenadas < ul>< /ul >:

  • Item 1
  • Item 2
  • Item 3

Uma lista dentro da outra:

  1. Item 1
  2. Item 2
  3. Item 3
    • Item 3.1
    • Item 3.2
    • Item 3.3
